Casa

A simple tool for configuring your developer environment.

$ ruby -e "$(curl --user [YOUR_GITHUB_USER_NAME] https://raw.githubusercontent.com/sixsixteen/casa/master/lib/casa/installer.rb)"
Github Documentation
Why Casa?

Having a development environment that you're comfortable with is like being in a nice, familiar space - a home. Likewise, trying to code in a different environment is distracting, frustrating, and makes coding less fun! Organizing a developer environemnt to fit your needs, from dotfiles to brew packages to custom scripts, allows you to understand your tools better and write better code.

Casa is a tool to help you configure you environment and run a simple command on any new machine so that it feels like you're right at home. Checkout the documentation and the Github repository!

How it works

There are several commands built into Casa (see them with $ casa help!) but the most important one is $ casa furnish.

This command, when run in a directory that containes a casa.yml file, builds out your environment exactly how you specified in your yaml file. It does so in a way that is completely system safe as well, everything that Casa does to your environment can be undone by simply uninstalling Casa.

What it looks like?

When configuring your environment configuration with Casa you'll need a directory structure with a casa.yml

# casa.yml

name: garrett's config
repo: https://github.com/casa-core/casa.git
type: config
brew_kegs:
  - imagemagick
  - ccrypt
ruby_gems:
  - launchy
pip_packages:
  - saf
node_modules:
  - graphicsmagick
casa_modules:
  - essentials
scripts:
  - link-dotfiles.sh

A more fleshed out configuration can contain two directories in addition to the casa.yml file:

  1. files which can be an level of nested files that you want your scripts to reference
  2. scripts to be executed in the order specified in your casa.yml. These scripts can reference your own files or powerful casa modules. Currently Casa supports bash, node, ruby, and python scripts.

Reference the docs to learn about modules and see how to take full advantage of Casa configurations!

my_config_dir/
  casa.yml
  files/
    dotfiles/
      .vimrc
      .bashrc
  scripts/
    link-dotfiles-with-system.sh
